## Issues Addressed

### Bug 1: License Progress Calculation Error
**Severity:** High  
**File:** `src/components/LicensePanel.vue`

**Problem:** License progress bar and upgrade availability check use `store.qsos` (current spendable balance) instead of `store.totalQsosEarned` (lifetime earned). This causes the progress bar to decrease when players buy factories, even though license unlocks should be based on total lifetime QSOs earned.

**Expected Behavior:**
- Player earns 60M QSOs → Progress shows 60M/50M (100%, ready for General)
- Player spends 20M on factories → Progress still shows 60M/50M (100%)
- Current buggy behavior: Progress shows 40M/50M (80%)

**Fix:**
- Line 62: Change `Number(store.qsos)` → `Number(store.totalQsosEarned)`
- Line 74: Change `store.qsos >= nextLicenseCost.value` → `store.totalQsosEarned >= nextLicenseCost.value`

**Testing:**
- Update LicensePanel.test.js to use totalQsosEarned in test data
- Verify progress uses lifetime earned, not current balance

---

### Bug 2: Missing Migration Notification UI
**Severity:** High  
**Files:** New component + App.vue

**Problem:** When v1.0.0 save data is detected, the game silently wipes progress and starts fresh. Users have no feedback about what happened or why their progress disappeared.

**Solution:** Create MigrationNotification component

**Component Spec:**
```
File: src/components/MigrationNotification.vue
Props: None (uses store.migrationInfo)
Trigger: Display when store.migrationInfo !== null
Features:
  - Modal/overlay design with terminal theme
  - Shows: "Game Updated" title
  - Shows: Migration from v1.0.0 to v1.1.1
  - Shows: Reason ("Major update requires fresh start")
  - Shows: Summary of old progress (factories count, QSOs, license level)
  - "Continue" button to dismiss
  - Auto-dismisses on click outside
```

**Integration:**
- Import into App.vue
- Place in template near top (before or after ErrorBoundary)
- Connect to store.migrationInfo reactive state

**Testing:**
- Manual test: Simulate v1.0.0 save by removing version field from localStorage

---

### Feature: Refresh Page Button
**Severity:** Medium  
**File:** `src/components/SettingsPanel.vue`

**Problem:** Mobile users in web app view often don't have access to browser refresh controls (no Ctrl+R, no refresh button). Additionally, mobile audio context sometimes gets stuck in a muted state and needs reinitialization.

**Solution:** Add refresh/reload button to settings

**Button Spec:**
```
Location: SettingsPanel.vue (Maintenance/Advanced section)
Text: "🔄 Refresh Page"
Action: window.location.reload()
Behavior:
  - Saves current game state before reload
  - Full browser refresh (reinitializes all services)
  - Reloads from localStorage (preserves all progress)
  - Fixes stuck audio context on mobile
```

**UX Considerations:**
- Place in "Advanced" or "Troubleshooting" section
- No confirmation needed (data is saved to localStorage)
- Add brief explanation: "Reload the game if audio stops working or UI becomes unresponsive"

## Technical Notes

### State Management
- `qsos` - Current spendable balance (decreases when buying)
- `totalQsosEarned` - Lifetime XP (never decreases, used for licenses)
- `migrationInfo` - Set when old save detected, cleared after displayed

### Mobile Audio Context
Mobile browsers suspend AudioContext when not actively used. The refresh button provides a way to:
1. Force new AudioContext creation
2. Re-request audio permissions if needed
3. Clear any stuck audio service state

### Migration Data Structure
```javascript
migrationInfo: {
  fromVersion: '1.0.0',
  toVersion: '1.1.0',
  reason: 'Major update requires fresh start',
  oldFactories: 15,
  oldQsos: '12345678',
  oldLicense: 2
}
```
